# 🤝 贡献指南

感谢你愿意完善 Skills 工具箱！

---

## 📝 文档规范

每个 Skill 文档需包含以下章节：

### 必须包含

| 章节 | 说明 | 示例 |
|------|------|------|
| **🎯 痛点 & 问题** | 具体解决什么问题，不要泛泛而谈 | "当代码出 Bug 时，AI 容易凭直觉乱猜修复，浪费大量时间在无效尝试上" |
| **⚙️ 原理** | 至少说明依赖什么工具/接口/机制 | "4 阶段根因调试法：调查→分析→假设→验证" |
| **📖 使用指南** | 含安装命令和至少一个使用示例 | `npx skills add ...` |
| **✅ 适用场景** | 什么时候该用这个 Skill | "测试失败、Bug 修复、生产异常排查" |
| **❌ 不适用场景** | 什么时候不该用、有什么坑 | "架构设计讨论不适用（应找技术架构师）" |

### 可选包含

| 章节 | 说明 |
|------|------|
| **🔗 相关链接** | GitHub 地址、原始文档 |
| **📝 更新日志** | 文档修改记录 |

## 🔀 贡献流程

### 方式一：完善现有文档

1. 在 `docs/` 目录找到对应的 `.md` 文件
2. 补充"痛点 & 原理"等空白部分
3. 提交 PR

### 方式二：新增 Skill

1. 在 `docs/` 目录创建新文件，命名格式：`序号-skill名称.md`
2. 使用上方模板编写
3. 更新 `docs/index.md` 中的分类索引
4. 提交 PR

### 方式三：新增分类

1. 在 `docs/` 创建新分类目录
2. 修改 `scripts/convert.js` 中的分类规则（如有自动化需求）
3. 更新本贡献指南
4. 提交 PR
