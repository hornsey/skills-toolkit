# validate-idea

> 创业想法验证助手 — 在投入时间之前验证商业想法的价值

## 🎯 痛点 & 问题

- **想法难以落地**：有创业想法但不知道是否值得追求
- **盲目投入风险**：花费数月开发产品后发现没人需要
- **闭门造车困境**：埋头做产品却不了解市场和用户真实需求
- **验证方法缺失**：不知道该如何系统性地验证想法的可行性
- **认知偏差**：创业者容易对自己的想法过度自信

这个 Skill 帮助解决以上问题，基于《极简创业者》的方法论，系统性地验证商业想法。

## ⚙️ 原理

基于 **Sahil Lavingia 的《The Minimalist Entrepreneur》** 方法论，核心思想是：

> **"在投入开发之前，先验证想法是否有人愿意付费"**

**验证框架（4步法）**：

1. **社区验证** — 找到你的目标用户群体，了解他们的痛点
2. **问题验证** — 确认这是一个真实存在的问题，而非假设
3. **解决方案验证** — 用最小成本提供解决方案，观察用户反应
4. **付费意愿验证** — 真正收到钱才是验证的最终标准

**核心理念**：
- **先卖后做**：先销售产品，再开发产品
- **手工流程优先**：先用人工方式提供服务，验证需求后再自动化
- **最小化可行产品**：MVP 不是最简产品，而是验证假设的最小实验

**与其他 Skill 的关系**：
```
/find-community → /validate-idea → /mvp → /processize → /first-customers
```
这是《极简创业者》旅程的第二步，第一步是找到社区。

### 使用示例

**验证创业想法：**
```
/validate-idea
```

**引导式验证：**
```
"请使用 validate-idea skill，帮我验证我的想法：一个面向独立开发者的自动化部署平台"
```

**具体问题验证：**
```
"我想做一款 AI 写作工具，先帮我验证：1) 目标用户是谁？2) 他们的痛点是什么？3) 愿意付费吗？"
```

### 验证问题清单

Skill 会引导你思考以下问题：
- 你的目标用户是谁？（具体到人）
- 他们现在用什么方式解决这个问题？
- 你能比现有方案好 10 倍吗？
- 如何找到你的前 10 个用户？
- 他们愿意付多少钱？为什么？

## ✅ 适用场景

- **创业初期验证**：有创业想法，需要系统性地验证可行性
- **产品方向选择**：在多个想法中决定做什么
- **市场调研辅助**：了解目标用户的真实需求和痛点
- **MVP 范围界定**：验证核心假设后确定最小可行产品范围
- **投资决策参考**：在投入大量资源前做最后一道检查

## ❌ 不适用场景

- **已经验证的想法**：已经有用户和数据支撑的方向
- **需要技术实现**：此 Skill 专注于验证，不包含开发指导
- **大规模市场分析**：需要专业市场调研报告的场景
- **快速执行导向**：已经决定要做，需要立即行动的阶段
- **企业内部创新**：更适合使用 `/processize` 技能

## ⚠️ 局限 & 风险

- **验证不等于成功**：即使通过验证，产品仍可能失败
- **认知偏差**：AI 辅助验证无法完全消除人类的认知偏差
- **市场变化**：验证时的市场状况可能快速变化
- **过度依赖风险**：可能让人过度依赖框架而缺乏独立思考
- **文化差异**：《极简创业者》的方法论主要基于美国 SaaS 创业环境
