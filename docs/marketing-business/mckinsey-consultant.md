# mckinsey-consultant

> 麦肯锡顾问 Skill —— 用 McKinsey 风格的问题定义、假设驱动分析与故事线表达来处理复杂商业问题

## 🎯 痛点 & 问题

商业问题真正难的地方，往往不是“缺一个答案”，而是缺一套能收敛复杂性的工作方法：

- **问题看起来很大，但无从拆解**：市场、竞争、组织、产品、财务纠缠在一起
- **分析逻辑不成体系**：知道要调研，却不知道先验证什么、后验证什么
- **报告像信息堆砌**：有数据、有观点，但缺少结论导向和故事线
- **PPT 很像笔记，不像咨询稿**：页面结构杂乱，无法支撑管理层决策
- **缺少假设驱动思维**：容易陷入无边界收集信息，而不是围绕关键判断做验证

`mckinsey-consultant` 的核心价值，是让 Agent 按咨询方法论工作：先定义问题，再搭树、列假设、收证据、形成结论，最后输出能拿去沟通的结构化材料。

## ⚙️ 原理

现有资料显示，该 Skill 以 **McKinsey Problem Solving** 方法论为核心，强调从问题定义到汇报输出的一整套链路。

### 核心机制

1. **问题定义优先**
   - 先把“到底要回答什么”说清楚，而不是一上来就做市场调研

2. **MECE 问题树拆解**
   - 通过 Issue Tree 把复杂问题拆成互斥且穷尽的子问题
   - 有助于避免分析重复、遗漏和逻辑混乱

3. **假设驱动分析**
   - 先提出可能成立的关键判断，再围绕它搜集数据
   - 比“把所有资料都看一遍”更节省时间，也更接近咨询实战

4. **故事线与页面结构输出**
   - 不只是分析结论，还强调如何把结论做成可沟通的汇报页面
   - 适合管理层汇报、投资分析、战略建议书等场景

### 典型流程

```text
定义问题 → 构建 Issue Tree → 提出核心假设 → 设计调研计划
→ 收集并分析证据 → 提炼 key findings → 组织故事线 → 生成咨询式 PPT / 报告
```

### 使用示例

#### 场景 1：市场进入研究

```text
用户：我们想进入企业知识管理市场，先帮我判断是否值得做。
AI：我会先定义核心决策问题，再用问题树拆分市场空间、竞争格局、切入点和商业化假设。
```

#### 场景 2：竞争分析

```text
用户：请分析我们和竞品相比最大的战略差距是什么。
AI：我会用假设驱动方式，围绕定位、渠道、产品能力和护城河进行结构化比较。
```

#### 场景 3：管理层汇报

```text
用户：把这份调研整理成领导能看懂的结论。
AI：我会优先抽出结论、依据与建议动作，再转成更像咨询汇报而不是研究笔记的表达方式。
```

## ✅ 适用场景

- **商业尽调**：投资前市场、竞争和商业模式验证
- **战略规划**：帮助团队定义方向、优先级与资源配置逻辑
- **市场进入研究**：评估新行业、新地区或新产品线机会
- **竞争分析**：对手对比、差距识别、赢法梳理
- **管理层汇报准备**：把分析结果组织成更专业的结论表达

---

## ❌ 不适用场景

- **只问简单事实**：不需要整套咨询流程的任务
- **即时交易型决策**：分钟级、秒级判断更依赖实时数据系统
- **纯创意发散**：该 Skill 偏收敛分析，不是开放式脑暴工具
- **没有问题边界的任务**：若连核心问题都没界定，容易导致输出过大而失焦

---

## ⚠️ 局限 & 风险

1. **方法论强，前置澄清成本也高**：比直接给建议更花时间
2. **数据质量决定上限**：如果证据来源薄弱，结构再好也难得出可靠结论
3. **咨询风格不等于事实正确**：“看起来专业”不能替代真实验证
4. **容易过度框架化**：有些创新问题不一定适合完全套进传统咨询分析框架
