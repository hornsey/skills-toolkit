# SoulTrace

> 性格测试 Skill —— 通过自适应人格测评，帮助 Agent 更理解用户的沟通风格、偏好与人格原型

**Source:** [GitHub - soultrace-ai/soultrace-skill](https://github.com/soultrace-ai/soultrace-skill)

**Platforms:** Claude Code / 兼容 SKILL.md 的 Agent 运行时 | **Assessment Model:** 5 色人格模型 + Bayesian Active Learning

---

## 🎯 痛点 & 问题

很多 AI 协作问题，本质上不是“回答不够聪明”，而是“没有真正理解人”：

- **沟通风格不匹配**：同样的建议，有人喜欢直接结论，有人需要背景和陪伴式解释
- **任务分工不顺**：团队协作或自我管理时，不同人格倾向会影响决策方式和执行方式
- **通用建议太泛**：没有用户画像支撑时，AI 很难给出真正个性化的互动方式
- **传统人格测试太机械**：题目多、耗时长，而且很难在 Agent 场景里自然接入

`SoulTrace` 的意义，在于让 Agent 通过更结构化的人格测评结果，调整沟通、陪伴、建议和任务协作方式。

---

## ⚙️ 原理

根据仓库 README 与官网公开信息，SoulTrace 是一套 **自适应人格评估** 方案。

### 核心机制

1. **5 色心理模型**
   - 以 5 色人格维度为基础
   - 用于描述个体在沟通、决策、节奏与价值偏好上的倾向

2. **Bayesian Active Learning**
   - 官网明确提到使用 Bayesian active learning
   - 这意味着问题不是静态死板地全部问完，而是根据前面的回答动态调整后续问题，更高效地逼近结果

3. **24 个问题 + 1-7 评分**
   - README 中说明，Agent 会引导用户回答 24 个问题
   - 采用 1-7 的 Likert 量表作答方式

4. **25 种人格原型输出**
   - 测评最终会归类为 25 种 archetype（人格原型）之一
   - 适合进一步作为沟通策略或协作建议的输入

### 工作流

```text
Agent 发起人格测评 → 用户完成 24 题量表回答
→ 系统基于 5 色模型与贝叶斯主动学习推断 → 输出 25 种人格原型之一
→ 用于后续沟通与协作建议
```

## ✅ 适用场景

- **自我认知**：希望更好理解自己的表达、决策与协作风格
- **职业发展辅导**：把人格画像作为建议个性化输入
- **团队协作改善**：辅助理解成员之间的沟通差异
- **陪伴型 Agent**：让 AI 在建议风格上更贴合用户
- **成长型对话场景**：结合教练、辅导、复盘类工作流使用

---

## ❌ 不适用场景

- **临床心理诊断**：人格测试不能替代专业心理咨询或医学评估
- **高风险用人决策**：不应把单次测评结果当成招聘、晋升或淘汰依据
- **只想快速拿结论**：若用户不愿认真完成问卷，结果参考价值会明显下降
- **绝对客观画像需求**：人格测评是辅助理解工具，不是对人的完整定义

---

## ⚠️ 局限 & 风险

1. **结果是辅助画像，不是标签真相**：人格原型适合帮助理解，不适合给人定性
2. **答题状态影响结果**：当下情绪、场景和自我认知偏差都会影响测评
3. **解释仍需谨慎**：AI 对结果的二次解读如果过度延伸，可能带来误导
4. **隐私敏感**：人格测评涉及个体心理与行为偏好，数据处理需注意隐私边界
