# OpenClaw Agent Optimize

> 优化你的 Agent —— 面向 OpenClaw 工作区的结构化审计与调优 Skill，聚焦成本、模型路由、上下文纪律、委派策略与可靠性

**Source:** [ClawHub - phenomenoner/openclaw-agent-optimize](https://clawhub.ai/phenomenoner/openclaw-agent-optimize)

**Type:** 咨询 / 审计型 Skill | **Platforms:** OpenClaw / ClawHub 生态

---

## 🎯 痛点 & 问题

很多 Agent 不是真的“能力不够”，而是运行方式不够健康：

- **成本失控**：所有任务都走高价模型，简单问题也按重型方案处理
- **模型路由混乱**：没有按任务类型选择模型，导致慢、贵、效果还不稳定
- **上下文膨胀**：每次都把历史和无关材料塞进上下文，推理成本越来越高
- **委派策略不清晰**：该并行的没并行，该拆分的没拆分，主 Agent 负担过重
- **工作区可靠性不足**：缺少检查、验证与回滚思路，容易改坏配置

`openclaw-agent-optimize` 的价值在于，它不是直接帮你乱改配置，而是先做结构化审计，再提出可回滚、可验证的优化方案。

---

## ⚙️ 原理

从 ClawHub 页面可提取到的公开描述来看，这个 Skill 的定位非常明确：

> 用于对 OpenClaw 工作区做结构化审计，给出“现状评估 → 可选方案 → 推荐计划 → 预期收益 → 回滚步骤”。

### 核心关注维度

| 维度 | 说明 |
|------|------|
| **Cost** | 成本控制与资源使用效率 |
| **Model Routing** | 按任务类型进行模型分流 |
| **Context Discipline** | 保持上下文精简，减少无效负担 |
| **Delegation** | 更合理地使用子任务与并行能力 |
| **Reliability** | 提升工作流稳定性、可验证性与恢复能力 |

### 默认工作姿态

ClawHub 页面强调：

- 这是一个 **advisory first** 的 Skill
- 默认输出应是：**审计结果、方案选项、推荐计划、精确变更提议、回滚方案、验证计划**
- **未经明确批准，不做持久性修改**

这点与一般“安装就改配置”的技能不同，属于偏稳健、偏生产环境的工作方式。

### 工作流

```text
审计当前 OpenClaw 工作区 → 识别成本/路由/上下文/委派/可靠性问题
→ 提出多个优化选项 → 输出推荐方案 → 明确变更点、验证方式与回滚路径
```

## ✅ 适用场景

- **OpenClaw 工作区治理**：希望系统性梳理当前 Agent 配置与工作流
- **成本优化**：模型费用、重复调用、上下文冗余等问题明显时
- **可靠性提升**：希望让复杂任务更稳定、可验证、可恢复
- **团队交付前整理**：准备将个人工作区转为团队共用规范时
- **需要方案而不是立刻动手改**：适合先评估、再实施的场景

---

## ❌ 不适用场景

- **要求立即修改配置**：这个 Skill 默认不是“直接改到位”，而是先给审计与方案
- **与 OpenClaw 无关的问题**：例如普通写作、代码生成、知识问答等
- **没有可检查的工作区上下文**：若没有实际工作区配置，审计价值有限
- **极短平快任务**：简单问题无需做完整优化评估

---

## ⚠️ 局限 & 风险

1. **公开资料有限**：当前仓库内能获得的是 Skill 卡片级说明，具体细节仍以实际 SKILL.md 为准
2. **建议质量依赖现状信息**：若当前工作区配置、任务模式、成本数据不清晰，优化建议会偏泛
3. **不应跳过审批**：文档明确偏向先建议后修改，擅自持久化改动可能违背 Skill 设计初衷
4. **优化不是一次性工作**：成本、模型与工作流会随团队使用方式持续变化
