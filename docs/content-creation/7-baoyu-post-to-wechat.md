## 🎯 痛点 & 问题

在将内容发布到微信公众号时，团队经常会遇到这些实际问题：

- **发布流程碎片化**：文案写在 Markdown，排版要去网页后台手工调整，图片和引用还要再次处理
- **外链兼容差**：普通网页链接直接贴到公众号里，阅读体验差，也容易丢失上下文
- **图文与文章双流程麻烦**：公众号“文章”和“贴图/图文”两种模式需要不同操作，人工切换费时
- **多人协作难复用**：运营、编辑、开发各自有自己的流程，缺少统一发布规范
- **自动化接入成本高**：直接对接公众号 API、浏览器脚本、鉴权和权限检查都不够友好

这个 Skill 的价值不只是“替你点按钮”，而是把“内容整理 → 格式转换 → 权限检查 → 发布执行”串成一条可复用的交付链路。

## ⚙️ 原理

根据 `baoyu-post-to-wechat` 的 SKILL.md，这个 Skill 围绕“公众号发布”封装了完整操作流。

### 核心机制

1. **多输入格式支持**
   - 支持 Markdown、HTML、纯文本作为输入
   - 适合把已有文章、日报、周报、教程等内容直接转成公众号可发布格式

2. **双发布路径**
   - **文章发布**：可通过 API 或浏览器方式发布公众号文章
   - **贴图 / 图文发布**：适合多图内容或更偏视觉流的公众号内容

3. **Markdown 转微信友好格式**
   - 提供 `md-to-wechat` 转换流程
   - 普通外链默认会转成底部引用，减少公众号正文中的跳转割裂感

4. **发布前检查**
   - 内置权限与环境检查脚本
   - 在真正发布前确认依赖、配置和可用性，避免运行到一半才发现缺权限

### 依赖工具 / 接口

| 组件 | 作用 |
|------|------|
| `bun` / `npx bun` | 运行仓库内脚本 |
| 微信公众号后台 / API | 真正完成文章或图文发布 |
| Chrome CDP / 浏览器自动化 | 在 API 不适用时走浏览器发布路径 |
| `scripts/check-permissions.ts` | 检查运行环境与权限 |
| `scripts/md-to-wechat.ts` | Markdown 转公众号格式 |
| `scripts/wechat-api.ts` | 通过 API 发布文章 |
| `scripts/wechat-browser.ts` | 通过浏览器流程发布贴图 / 图文 |
| `scripts/wechat-article.ts` | 通过浏览器流程发布文章 |

### 工作流

```text
原始内容 → 识别发布类型（文章/贴图）→ 检查权限与环境
→ 必要时做 Markdown 转换 → 选择 API 或浏览器发布路径 → 发布结果确认
```

## 📖 使用指南

### 安装方式

```bash
# 安装整个 baoyu-skills 仓库中的该 Skill
npx skills add https://github.com/jimliu/baoyu-skills --skill baoyu-post-to-wechat
```

如果当前环境没有 `bun`，通常需要额外准备：

```bash
# 先确保能运行 bun 或 npx bun
bun --version
# 或
npx -y bun --version
```

### 典型触发方式

```text
请把这篇 Markdown 文章发布到微信公众号
```

```text
使用 baoyu-post-to-wechat skill，把这个产品周报整理成公众号文章并发布
```

```text
把这组图片和说明文字发布成公众号贴图
```

### 使用示例

#### 场景 1：Markdown 文章发布

```text
用户：请把这篇产品更新日志发布到公众号。
AI：我会先检查当前环境和权限，再把 Markdown 转成微信公众号更友好的 HTML，最后选择 API 或浏览器发布路径执行。
```

#### 场景 2：已有排版内容直接发文

```text
用户：这段 HTML 已经排好版了，直接发公众号文章。
AI：我会跳过不必要的格式转换，重点检查图片、外链和发布权限后再执行。
```

#### 场景 3：多图图文发布

```text
用户：请把这 6 张活动海报和介绍发成公众号图文。
AI：我会走贴图 / 图文发布流程，整理图片顺序、说明文字和封面配置后再发布。
```

### 建议的发布前检查项

- 是否已准备好公众号后台权限
- 是否已配置对应 API 凭证（如果走 API）
- 图片链接是否可访问
- Markdown 中是否存在大量不适合公众号的外链
- 标题、摘要、封面图是否已确定

## ✅ 适用场景

- **技术博客同步到公众号**：把开发文档、周报、教程转成公众号文章
- **运营内容批量分发**：将日报、活动通知、产品动态统一发布到微信生态
- **AI 写作结果再加工**：先由其他 Skill 产出初稿，再用本 Skill 完成微信端交付
- **图文海报分发**：活动海报、知识卡片、长图内容的公众号图文发布
- **团队标准化运营流程**：把“写稿—排版—发布”固化为可复用操作规范

---

## ❌ 不适用场景

- **只需要普通网页发布**：如果不是发公众号，没必要引入微信特定流程
- **完全离线环境**：发布依赖公众号后台或 API，无法离线完成
- **高度定制视觉排版**：公众号复杂视觉稿仍可能需要人工微调
- **没有权限的代发需求**：没有后台权限或 API 凭证时，Skill 无法代替授权

---

## ⚠️ 局限 & 风险

1. **平台规则约束**：公众号后台规则、编辑器能力和接口权限会影响最终效果
2. **浏览器方案稳定性**：如果页面结构变化，浏览器自动化路径可能需要调整
3. **内容格式损耗**：Markdown 转换到公众号 HTML 过程中，复杂样式可能被简化
4. **敏感内容审核**：Skill 能完成发布流程，但无法替代平台内容审核结果
5. **账号安全风险**：涉及浏览器登录态或 API 凭证时，需要妥善保管账号权限
