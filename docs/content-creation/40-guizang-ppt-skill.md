# guizang-ppt-skill

> 将文章或主题快速生成为横向翻页网页 PPT 的展示型 Skill，强调设计感、节奏感和单文件交付

**License:** 以仓库为准 | **Platforms:** Claude Code / 兼容 SKILL.md 的 Agent 运行时 | **Output:** 单 HTML 横向翻页网页 PPT

## 🎯 痛点 & 问题

常见的“AI 做 PPT”工具，通常能快速出结构，却难同时兼顾视觉风格和交付便利：

- **普通 AI PPT 只会排版，不会设计**：内容有了，但缺少视觉节奏与展示张力
- **传统 PPT 文件协作麻烦**：字体、动画、版本兼容和导出常出问题
- **演讲内容与页面风格脱节**：文案和视觉不是同一套叙事逻辑，成品看起来像模板拼装
- **网页化展示门槛高**：想做可分享的 Web PPT，又要前端、设计和内容三套能力
- **风格难统一**：科技发布、行业分享、年度总结等不同主题，往往需要完全不同的表达方式

`guizang-ppt-skill` 的价值在于：不仅帮你“列提纲”，还把结构、模板、美术方向和最终交付形式打包成一个以演讲为中心的展示工作流。

## ⚙️ 原理

根据仓库中的 SKILL.md，这个 Skill 面向的是 **单文件 HTML 的横向翻页 PPT**，并且内置了两套明确的视觉体系。

### 核心机制

1. **单文件 HTML 交付**
   - 结果不是传统 `.pptx`，而是一个可直接打开、可分享的 HTML 演示文件
   - 更适合在线演示、网页托管、分享链接和个性化视觉展示

2. **双风格体系**

| 风格 | 特点 | 适合场景 |
|------|------|----------|
| **电子杂志 × 电子墨水** | 衬线标题、流体背景、暖色调、杂志感较强 | 人文分享、商业观察、行业洞察 |
| **瑞士国际主义（Swiss Style）** | 无衬线、强网格、功能色鲜明、信息密度高 | 科技发布、数据汇报、设计/工程分享 |

3. **模板化版式能力**
   - 支持章节封面、数据大字报、图片网格、信息页等常见演讲版式
   - 不是一页一页自由拼，而是先围绕展示叙事选择合适的页面模块

4. **网页演示交互**
   - 支持横向翻页
   - 可通过键盘、滚轮、触控等方式浏览
   - 更像一个“带节奏的网页演讲作品”而不是传统幻灯片

### 依赖工具 / 接口

| 组件 | 作用 |
|------|------|
| `SKILL.md` 主指令 | 规定风格、场景与输出要求 |
| `assets/template.html` / `template-swiss.html` | 两类网页 PPT 模板 |
| `references/themes*.md` | 主题色与视觉规范 |
| `references/layouts*.md` | 不同页面结构与版式说明 |
| Agent 本身 | 根据主题、受众和场景组织内容并填充模板 |

### 工作流

```text
主题 / 原文 / 目标场景 → 选择视觉风格 → 规划章节结构
→ 选择页面模块 → 生成单 HTML 网页 PPT → 本地预览与微调
```

```bash

```

```text

```

```text
使用 guizang-ppt-skill，把这份产品发布材料转成瑞士风横向翻页演示稿
```

```text
帮我做一份适合 Demo Day 的 Web PPT，强调视觉冲击力和章节节奏
```

### 使用示例

#### 场景 1：商业分享

```text
用户：把“2026 年 AI Agent 行业趋势”做成一份适合线下分享的网页 PPT。
AI：我会先按演讲场景规划章节，再判断用杂志风还是瑞士风，并输出横向翻页的 HTML 演示稿。
```

#### 场景 2：产品发布

```text
用户：把我们的新品说明做成科技感强一点的发布会演示。
AI：更适合瑞士国际主义风格，我会突出数据页、产品亮点页和节奏型章节封面。
```

#### 场景 3：文章改 PPT

```text
用户：把这篇长文变成 12 页以内的演示稿。
AI：我会先压缩信息密度，再把文章逻辑转成演示叙事，而不是逐段照搬正文。
```

### 使用前建议准备

- 演示用途（路演、汇报、分享、发布会）
- 受众是谁
- 希望偏向哪种风格
- 是否已有品牌色、Logo、配图
- 页数上限和演讲时长

## ✅ 适用场景

- **行业分享 / 内部汇报**：需要比传统 PPT 更有观感的展示形式
- **产品发布 / Demo Day**：强调页面张力和叙事节奏
- **文章转演示稿**：将长文、报告、研究材料转成可讲述内容
- **个人品牌表达**：希望成品更有作者风格而不是标准模板
- **网页化交付**：适合放到静态站点、在线链接或直接本地打开演示

---

## ❌ 不适用场景

- **必须交付 `.pptx` 的客户流程**：这个 Skill 的核心输出是 HTML，不是传统 PowerPoint 文件
- **极度保守的企业模板场景**：如果必须套用公司固定 PPT 模板，网页风格可能不匹配
- **高度表格化内容**：纯表格、纯数字台账类材料不一定适合这种展示方式
- **完全不会做基础校对**：生成稿通常仍需要人工审阅内容准确性和细节视觉

---

## ⚠️ 局限 & 风险

1. **输出格式限制**：核心产物是网页演示，不适合所有企业交付链路
2. **设计感强但需取舍**：视觉表达增强后，信息密度常需要主动压缩
3. **浏览器兼容性**：HTML 动效、WebGL 背景等效果在不同设备上表现可能不同
4. **品牌一致性问题**：若企业已有严格品牌规范，模板需进一步定制
5. **图片与版权风险**：演示中引用图片、图标或素材时需要确认授权
