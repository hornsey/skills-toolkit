**Platforms:** SkillHub / OpenClaw 等兼容 SkillHub 生态的运行时

## 🎯 痛点 & 问题

内容创作类任务往往不是单一步骤，而是一串连续动作：

- **不知道写什么**：选题、角度、结构都要从零开始
- **写得出来但不好发**：不同平台对篇幅、语气、排版要求差别很大
- **素材零散**：有会议纪要、录音、文章、热点，但难整理成成品
- **效率不稳定**：灵感来时写得快，没有框架时又会卡很久
- **中文语境适配难**：很多英文创作框架不完全适合中文传播场景

`Content Creator` 这类综合内容 Skill 的价值，是把“找选题、搭结构、写初稿、改表达、适配平台”打成一体化流程。

## ⚙️ 原理

当前仓库内可获得的公开信息主要来自分类页与 SkillHub 收录页名称，能确认其定位是“内容创作助手”。基于这一定位，这类 Skill 一般围绕以下机制工作：

### 典型能力模块

1. **选题与方向整理**
   - 根据主题、用户群体和目标平台，先明确内容方向

2. **结构化写作**
   - 先输出标题、提纲、小节逻辑，再进入正文创作

3. **平台适配**
   - 同一份素材可拆成公众号文、社媒短帖、视频口播稿或知识卡片文案

4. **语言润色**
   - 优化语气、节奏与可读性，让成品更适合中文传播环境

### 可能的工作流

```text
原始主题 / 素材 → 明确目标读者与平台 → 生成提纲
→ 输出初稿 → 做平台化改写 → 最终润色与发布准备
```

由于 SkillHub 页面当前可直接提取的信息有限，具体命令、参数与内部脚本仍应以实际安装后的 SKILL.md 为准。

### 使用示例

#### 场景 1：从想法到文章

```text
用户：我想写一篇关于 AI Agent 工作流的文章，但不知道怎么开头。
AI：我会先帮你确定目标读者和角度，再给出标题方案、提纲和初稿结构。
```

#### 场景 2：素材整理成成品

```text
用户：我有一段直播纪要，帮我整理成公众号文章。

```

#### 场景 3：多平台改写

```text
用户：把这篇长文拆成小红书文案和朋友圈短文。
AI：适合先用 Content Creator 做主稿，再拆成多个渠道版本。
```

## ✅ 适用场景

- **中文文章写作**：公众号、博客、知识型长文
- **选题策划**：围绕某个领域批量出内容方向
- **素材再生产**：把录音、纪要、长文重新整理成新内容
- **平台改写**：从长文拆成短帖、提纲、口播稿等
- **个人品牌 / 社群运营**：持续产出稳定内容

---

## ❌ 不适用场景

- **需要极强专业合规的内容**：法律、医疗、金融等仍需专业审核
- **只做单句文案打磨**：若只是改一句标题或 CTA，用更专门的文案 Skill 更直接
- **纯设计导向内容**：如果需求主要是海报、视频或漫画，不是它的强项
- **没有任何素材或目标边界**：完全无方向的任务会导致输出过泛

---

## ⚠️ 局限 & 风险

1. **公开资料有限**：当前项目内对该 Skill 的公开信息较少，具体行为仍以安装后指令为准
2. **综合型 Skill 容易泛化**：如果不给清晰受众、平台和目标，输出容易流于通用模板
3. **事实核验仍重要**：内容创作类 Skill 容易生成未经验证的信息或案例
4. **平台规则变化快**：不同内容平台的风格和限制会持续变化
